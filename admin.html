Déployer l'agent IA à distance

Comment mettre en ligne votre script Python pour un client, sans aller chez lui.

💡 Principe : Votre script Python tourne sur un serveur cloud (Railway ou Render). Le client y accède via un lien ou via WhatsApp. Vous gérez tout depuis votre ordinateur. Aucune installation chez le client.
1

Préparer votre script Python

Votre script doit exposer une API HTTP. Ajoutez Flask ou FastAPI :

pip install flask anthropic # Dans votre script : from flask import Flask, request, jsonify import anthropic app = Flask(__name__) client = anthropic.Anthropic(api_key="votre_cle_api") @app.route('/chat', methods=['POST']) def chat(): data = request.json message = data.get('message', '') # Votre logique ici response = client.messages.create( model="claude-sonnet-4-20250514", max_tokens=500, messages=[{"role": "user", "content": message}] ) return jsonify({"reply": response.content[0].text}) if __name__ == '__main__': app.run(host='0.0.0.0', port=8080)

Créez aussi un fichier requirements.txt avec : flask et anthropic

2

Déployer sur Railway (recommandé, gratuit au début)

  • Allez sur railway.app et créez un compte gratuit
  • Cliquez sur « New Project » → « Deploy from GitHub repo »
  • Uploadez votre script (ou connectez GitHub)
  • Dans les variables d'environnement, ajoutez : ANTHROPIC_API_KEY = votre_clé_api
  • Railway détecte Python automatiquement et démarre votre app
  • Vous obtenez une URL publique type https://votre-app.railway.app
⚡ Render.com est une alternative gratuite similaire. Allez sur render.com, créez un « Web Service » et uploadez votre code.
3

Personnaliser par client

Pour chaque client, créez un déploiement séparé avec ses infos :

  • Ajoutez une variable d'env CLIENT_NAME, CLIENT_INFO (horaires, FAQ, etc.)
  • Ou créez un fichier config_client.json par client dans votre repo
  • Chaque client = un projet Railway séparé = une URL séparée
4

Donner accès à l'agent au client

Une fois déployé, le client peut interagir avec son agent de 3 façons :

  • Lien direct : Vous lui donnez l'URL Railway. Il peut l'intégrer sur son site ou partager le lien.
  • Widget HTML : Vous lui fournissez un petit snippet HTML/JS à coller sur son site qui appelle votre API.
  • WhatsApp : Vous connectez l'URL Railway à WhatsApp Business API (voir section WhatsApp).
⚠️ Important : Ne donnez jamais votre clé API Anthropic directement au client. Stockez-la toujours côté serveur (variables d'environnement Railway).
5

Sécuriser l'accès (optionnel mais recommandé)

Pour éviter que n'importe qui utilise votre API :

# Ajoutez un token secret dans les headers import os SECRET_TOKEN = os.environ.get('SECRET_TOKEN', 'mon-token-secret') @app.route('/chat', methods=['POST']) def chat(): token = request.headers.get('X-API-Token') if token != SECRET_TOKEN: return jsonify({"error": "Non autorisé"}), 401 # ... reste du code

Chaque client a son propre token. Vous le changez en modifiant la variable d'env dans Railway.

Intégration WhatsApp

Comment faire répondre votre agent IA automatiquement sur WhatsApp.

📱 L'idée : Un client envoie un message WhatsApp → WhatsApp transmet le message à votre script Python → Votre script demande à Claude de répondre → La réponse repart sur WhatsApp automatiquement.
Deux options disponibles

Option A — WhatsApp Business API officielle (recommandé à terme)

Payante après un certain volume, mais officielle et stable. Nécessite un numéro de téléphone dédié.

Option B — Twilio WhatsApp (plus simple pour commencer)

Twilio propose un bac à sable (sandbox) WhatsApp gratuit pour tester. Idéal pour démarrer.

👇 Option B (Twilio) — Le chemin le plus rapide :

1

Créer un compte Twilio

  • Allez sur twilio.com et créez un compte gratuit
  • Dans le dashboard, allez dans Messaging → Try it out → Send a WhatsApp message
  • Suivez les instructions pour rejoindre le sandbox (vous envoyez un code par WhatsApp)
  • Récupérez votre Account SID et Auth Token dans le dashboard
2

Modifier votre script Python

Ajoutez la librairie Twilio et créez un webhook :

pip install twilio from twilio.rest import Client as TwilioClient from twilio.twiml.messaging_response import MessagingResponse TWILIO_SID = os.environ.get('TWILIO_SID') TWILIO_TOKEN = os.environ.get('TWILIO_TOKEN') TWILIO_FROM = 'whatsapp:+14155238886' # Numéro sandbox Twilio @app.route('/whatsapp', methods=['POST']) def whatsapp_webhook(): incoming_msg = request.form.get('Body', '').strip() sender = request.form.get('From', '') # Demander à Claude de répondre response_claude = client.messages.create( model="claude-sonnet-4-20250514", max_tokens=500, system="Tu es l'assistant IA du commerce XYZ...", messages=[{"role": "user", "content": incoming_msg}] ) reply_text = response_claude.content[0].text # Renvoyer sur WhatsApp resp = MessagingResponse() resp.message(reply_text) return str(resp), 200, {'Content-Type': 'text/xml'}
3

Connecter Twilio à votre URL Railway

  • Déployez votre script mis à jour sur Railway
  • Dans Twilio Dashboard → WhatsApp Sandbox → « When a message comes in »
  • Entrez : https://votre-app.railway.app/whatsapp
  • Méthode : HTTP POST
  • Cliquez Save
4

Pour un numéro WhatsApp réel (client final)

Le sandbox Twilio utilise un numéro partagé. Pour un numéro dédié au client :

  • Option 1 : WhatsApp Business API via Meta → business.whatsapp.com. Nécessite une approbation Meta (1-2 semaines).
  • Option 2 : Twilio payant avec un numéro WhatsApp approuvé (~25$/mois)
  • Option 3 : 360dialog (partenaire WhatsApp plus simple) → 360dialog.com
⚠️ Pour la phase de lancement, le sandbox Twilio suffit amplement pour faire des démos et tester avec vos premiers clients.
5

Sécurité du panel admin

Informations sur la protection de votre accès admin.

Protections actives

Mécanismes de sécurité en place

  • ✅ Authentification en 2 étapes (mot de passe + PIN)
  • ✅ Mots de passe hashés en SHA-256 (jamais stockés en clair)
  • ✅ Blocage temporaire après 5 tentatives échouées (15 minutes)
  • ✅ Session expirante automatiquement après 30 minutes d'inactivité
  • ✅ Redirection automatique si accès direct sans token
  • ✅ Token de session unique généré à chaque connexion
Comment changer vos identifiants

Modifier le mot de passe ou le PIN

Pour changer vos identifiants, vous devez générer un nouveau hash SHA-256. Procédure :

  1. Ouvrez n'importe quelle page de votre site dans Chrome/Firefox
  2. Ouvrez la console développeur (F12 → onglet Console)
  3. Tapez la commande suivante (en remplaçant "VotreNouveauMotDePasse") :
await crypto.subtle.digest('SHA-256', new TextEncoder().encode('VotreNouveauMotDePasse')) .then(b => Array.from(new Uint8Array(b)).map(x=>x.toString(16).padStart(2,'0')).join(''))
  1. Copiez le résultat (une longue chaîne de caractères)
  2. Ouvrez admin-login.html dans un éditeur
  3. Remplacez la valeur de ADMIN_PWD_HASH ou ADMIN_PIN_HASH

Limites importantes à connaître

Ce que cette protection ne couvre pas

  • Un fichier HTML statique reste lisible par n'importe qui avec un éditeur de texte — ne stockez jamais de vraies données sensibles (numéros bancaires, clés d'API) dans ces fichiers.
  • Les données clients sont dans le localStorage du navigateur — si quelqu'un a accès physique à votre ordinateur, il peut les lire.
  • Pour une vraie sécurité production, hébergez sur un vrai serveur avec authentification backend (Node.js + JWT, ou Firebase Auth).
